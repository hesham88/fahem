Unlike

1.2  Shifting  from  Prototype  to  Production-Shaped  Projects  While  the  --prototype  flag  is  ideal  for  rapid  experimentation,  production  readiness  requires  a transition

to

"full-scale"

Using

enhance

upgrades

prototypes

into

enterprise-ready

projects

by

generating

72

critical

assets.

These

include

Terraform

files

for

Infrastructure

as

Code

(IaC),

GitHub

Actions

or

Cloud

Build

workflows,

and

deployment

manifests

for

containerized

environments.Crucially,

production-shaped

projects

utilize

Collaboration

Modes

to

the

agent's

scope

and

behavior: ●  Chat  (Default):   Full  multi-turn  user  interaction.

●  Task:   Focused  on  specific  goals  with  automatic  return  to  parent  agents  upon completion. ●  Single-turn:   No  user  interaction  allowed;  designed  for  parallel  execution  and  immediate results.This

structured

ensures

2.  Multi-Level  Authentication  and  Security  Architecture  Security  in  agentic  systems  is  predicated  on  tiered  authentication.  By  separating  the  identities used

for

development,

model

interaction,

and

infrastructure

provisioning,

architects

can

prevent

unauthorized

resource

usage

and

strictly

limit

the

"blast

radius"

of

a

potential

compromise.

2.1  Tiered  Authentication  Taxonomy  The  ADK  framework  employs  a  three-level  authentication  strategy:  ●  Level  1  (Coding  Agent  Auth):   Credentials  for  the  developer’s  coding  assistant  (e.g., Gemini

CLI

uses

gcloud

auth

login,

while

Claude

Code

utilizes

an

Anthropic

account

or

API

key). ●  Level  2  (Model  Auth):   Authenticates  the  agent's  calls  to  the  LLM.  ●  Gemini  API  Keys:   Best  for  local  development  via  AI  Studio;  does  not  support cloud-native

deployment

features. ●  Vertex  AI  (Application  Default  Credentials):   Required  for  enterprise-grade  features, high-scale

model

access,

and

production

environments. ●  Level  3  (Deployment  Auth):   Uses  ADC  to  unlock  provisioning  capabilities,  including agents-cli

deploy,

Terraform

state

management,

and

Datastore

RAG

provisioning.

2.2  Managed  Identities  and  Least  Privilege  For  production,  architects  must  use  the  --agent-identity  flag  during  deployment  to  assign  a unique

Google

Cloud

Service

Account

to

agent.

This

enforces

least

privilege:

the

agent’s

to

APIs

This

at

infrastructure

level,

providing

fallback

if

an

agent's

reasoning

fails.Establishing

these

secure

4.  CI/CD  Pipeline  Architecture  and  Governance  Gates  In  autonomous  systems,  the  CI/CD  pipeline  is  the  final  gatekeeper.  It  ensures  that  changes  to prompts,

4.1  The  Three-Stage  Deployment  Pipeline  and  Quantified  Evaluation  The  framework  supports  a  high-governance  workflow:  1.  CI  (Pull  Request):   Executes  unit  and  integration  tests.  Architects  should  expect   5-10+ iterations

of

for

merge. 2.  Staging  CD  (Merge  to  Main):   Builds  container  images,  pushes  to  Artifact  Registry,  and performs

automated

load

the

identical

container

image

validated

in

staging,

ensuring

binary

parity

across

environments.

5.2  Solving  Real-World  Operational  Challenges  Architects  mitigate  four  primary  failure  modes  via  production  telemetry:  ●  Hallucinated  Remediation:   Mitigation  via  the  tool_trajectory_avg_score  rubric, comparing

production

BigQuery
